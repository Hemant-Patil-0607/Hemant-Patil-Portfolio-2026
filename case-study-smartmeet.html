04 — Engineering Decisions

Key technical decisions that shaped the product.

Multi-LLM Support

Integrated OpenAI GPT, Anthropic Claude, and Google Gemini to provide flexibility and redundancy in AI processing.

Real-time Processing

Used WebSockets for live transcription feedback and FastAPI for high-performance async backend workflows.

Modular AI Workflows

Designed composable pipeline stages that can be rearranged or extended without breaking existing flows.

Structured Output Design

Enforced consistent JSON schemas for AI outputs to ensure reliable downstream processing and integration.

05 — Challenges

What was difficult.

Variable Input Quality

Meeting recordings vary wildly in audio quality, speaker overlap, and language. Building resilient transcription pipelines required extensive testing and fallback strategies.

AI Output Consistency

LLMs don't always produce structured output. Designing prompt engineering strategies and validation layers to ensure reliable, parseable results was a core challenge.

Real-time Performance

Processing large audio files while maintaining responsive UI required careful orchestration of async tasks, WebSocket communication, and background processing.

06 — Solution

How we solved it.

Retry & Validation Layer

Built automated validation pipelines that check AI outputs against expected schemas and retry with adjusted prompts when needed.

Progressive Processing

Implemented chunked audio processing with progress indicators, allowing users to see real-time transcription progress.

Abstraction Layer

Created a unified LLM interface that abstracts provider differences, making it easy to switch or add new AI providers.

07 — Outcome

What now works.

✓ End-to-end meeting intelligence from audio upload to structured reports
✓ Multi-provider AI integration with GPT, Claude, and Gemini
✓ Automated action item extraction and task generation
✓ Real-time transcription with WebSocket-powered progress
✓ Modular workflow engine for downstream AI automation
